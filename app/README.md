# app — menu-bar приложение (Фаза 2, для себя)

Иконка в строке меню: запись встречи в один клик + транскрипт по кнопке.
Захват — ScreenCaptureKit (системный звук + микрофон). Транскрипт — локальный
`transcribe.py` через venv (полное качество, ничего не уходит в облако).

## Запуск

```bash
cd app
swift run MeetingRecorder     # или ./run.sh
```

В строке меню появится иконка микрофона. Клик → окошко с кнопками.

Первый запуск запросит **Screen & System Audio Recording** и **Microphone** —
дай оба (Системные настройки → Приватность и безопасность), перезапусти.

## Как пользоваться

1. Клик по иконке в строке меню → **Запись**.
2. Проведи встречу (звук любого приложения + твой микрофон пишутся).
3. **Стоп** → транскрипция стартует **автоматически** (~минута).
4. Готовый транскрипт лежит текстовым файлом в
   **`~/Documents/Транскрипты встреч/<дата_время>.md`**.
   В окне — кнопки «Показать в Finder» и «Папка транскриптов».

Окно не показывает текст — только статус. Сырые дорожки и исходный
`transcript.md` остаются в `mac-capture/recordings/<timestamp>/`.

## Зависимости

- Собранный venv в корне проекта (`../.venv`) с `transcribe.py` — см. корневой README.
- Модель whisper в кэше (`python prefetch.py`) — иначе первый транскрипт будет
  долго качать модель.

## Календарь (всплывашка перед встречей)

Если настроен Google Calendar (см. корневой README раздел «Календарь» —
`credentials.json` + `python calendar_watch.py --auth`), аппка раз в 30с
опрашивает ближайшие встречи и за минуту до старта показывает плавающее
окошко: **«Подключиться и записать»** (откроет Zoom/Meet + начнёт запись) /
**«Только записать»** / **«Позже»**.

В окне аппки видно следующую встречу + кнопка **«Проверить напоминание»** —
показывает всплывашку сразу, без ожидания (для проверки UI).

Без настроенного календаря блок просто не появляется — аппка работает как
обычный рекордер.
